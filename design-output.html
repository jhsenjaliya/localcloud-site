Quick Start

One command to run them all.

Terminal
# Pull and start LocalCloud
$ docker pull jaysen2apache/localcloud
$ docker run -d \
-p 8080:8080 -p 4443:4443 \
-p 8085-8087:8085-8087 \
-p 9010:9010 -p 9050:9050 -p 6379:6379 \
-m 4g --name localcloud jaysen2apache/localcloud
# Configure your SDKs
$ eval "$(curl -s localhost:8080/_localcloud/env?format=shell)"
✓ Ready. Your GCP code works locally now.
Why LocalCloud

Like LocalStack for AWS. But for Google Cloud.

Env Vars Only

Zero Code Changes

Your existing GCP SDK code connects to localhost via standard emulator environment variables. Unset them to return to production.

< 60s Boot

Single Container

One docker run starts all services. No compose sprawl, no port conflicts. First API call within 10 minutes of install.

No Cloud

Fully Offline

No GCP credentials. No internet. No service accounts. Develop on a plane, at a coffee shop, wherever you write code.

60%+ Savings

Kill Cloud Dev Costs

Eliminate always-on cloud dev environments. No more surprise bills from forgotten Spanner instances or runaway BigQuery scans.
